# 数据来源实践：模型之前先问数据从哪里来

数学建模里，数据和模型一样重要。很多解法的差距不在于算法名字，而在于它到底使用了什么数据、这些数据是否可靠、是否可复现、是否支撑后续结论。

在 baseline、advanced、outstanding 三层解法里，数据来源通常也会逐步升级：

| 层级 | 数据状态 | 典型做法 |
|---|---|---|
| Baseline | 题面文字、少量参数、通用示例表 | 先把变量和模型跑通 |
| Advanced | 官方附件、官方数据、可复查的本地文件 | 用真实数据标定参数、生成结果和图表 |
| Outstanding | 官方数据 + O 奖论文表格/参数/报告结果 + 必要外部数据 | 复现论文关键结果，并清楚区分数据来源 |

## 一、数据来源的三种类型

### 1. 竞赛官方数据

这是最优先使用的数据来源，包括题面 PDF/HTML/docx、官方 Excel/CSV 附件、示例结果表，以及题面明确给出的参数、约束和单位。

优点是权威、可复查、最符合评委预期，也不容易引入额外争议。

典型写法：

```text
本文优先使用竞赛官方附件中的材料参数、实测温度、企业发票和 NFLIS 数据。
所有计算结果均由官方附件读取并生成，保证可复现性。
```

### 2. 外部公开数据

当官方数据不足以支撑模型时，需要自己找外部数据。

常见来源包括 WHO、CDC、World Bank、政府统计网站、地理边界、道路网络、人口、距离、医疗资源、行业报告、交通流量和事故统计。

使用外部数据时要特别小心：

- 数据时间范围是否和题目一致。
- 地理范围是否一致。
- 单位是否一致。
- 数据是否有缺失或统计口径变化。
- 是否需要在论文中说明来源和处理方式。

典型写法：

```text
由于题面未提供完整疫情时间序列，本文补充 WHO 公布的病例数据，并统一到日尺度后用于参数估计。
```

### 3. O 奖论文中的表格、参数和报告结果

Outstanding 复现经常会用到获奖论文中的表格、参数和结论。

这类内容不是竞赛原始附件，而是获奖论文作者处理后的结果，例如论文中的初始状态表、拟合参数、最优厚度、路线、分配结果和敏感性分析表。

使用时要标清：

```text
该参数来自 O 奖论文报告结果，不是题面原始附件。
```

否则容易把“论文复现”误写成“官方数据直接计算”。

## 二、当前 6 道样例题的数据情况

| 赛题 | Baseline/Advanced 数据 | Outstanding 数据 | 是否需要外部数据 |
|---|---|---|---|
| MCM 2015-A Ebola | COMAP 官方 HTML 题面 | 35532 论文 Table 7 初值、流行病参数、医疗中心覆盖结论 | 若完整复现真实疫情曲线，需要 WHO/CDC/人口/距离等外部数据 |
| CUMCM 2018-A 高温服 | 官方附件材料参数和实测温度 | 官方附件 + A466 论文换热系数和厚度结果 | 基本不需要外部数据 |
| MCM 2017-B Toll Plaza | 官方题面参数 | 69427 论文 Florida 417 校准值、事故率表、NJ 应用结果 | 若独立校准，需要外部交通流量、服务率、事故数据 |
| CUMCM 2020-B 穿越沙漠 | 官方附件、天气/关卡参数、Result.xlsx | 官方附件 + B108 路线表、情景枚举和博弈结论 | 基本不需要外部数据 |
| MCM 2019-C Opioid | COMAP 官方 NFLIS + ACS 数据 | 官方 NFLIS/ACS + 1901213 空间传播结果 | 若做更精细空间传播，可补县邻接或地理 shapefile |
| CUMCM 2020-C 信贷 | 官方三份 Excel 附件 | 官方附件 + C227 分类器和 soft voting 结果 | 基本不需要外部数据 |

## 三、哪些题官方数据已经足够

官方数据基本足够的题：

- CUMCM 2018-A 高温作业服。
- CUMCM 2020-B 穿越沙漠。
- MCM 2019-C Opioid Crisis。
- CUMCM 2020-C 中小微企业信贷。

这些题的 advanced 和 outstanding 都可以主要围绕官方附件或官方数据做。区别在于：

- Advanced 更强调自己读取附件并建模型。
- Outstanding 更强调复现 O 奖论文中的关键结果和中间表。

## 四、哪些题更依赖外部数据

| 赛题 | 为什么需要外部数据 |
|---|---|
| MCM 2015-A Ebola | 题面没有完整疫情时间序列、国家医疗资源、距离矩阵和药品生产真实数据 |
| MCM 2017-B Toll Plaza | 题面只有抽象要求和示例参数，真实服务率、交通流量、事故率、道路面积需要外部校准 |
| MCM 2019-C Opioid | 官方数据够做基础模型，但精细空间传播最好补县邻接、地理距离或 shapefile |

重要原则：

```text
如果外部数据只是让论文更丰富，但不是核心结论必需，可以作为增强；
如果没有外部数据就无法估计关键参数，则必须明确说明数据来源和假设。
```

## 五、论文写作时如何交代数据

建议每篇论文都单独写一个数据来源表。

| 数据 | 来源 | 用途 | 是否官方 | 处理方式 |
|---|---|---|---|---|
| 材料参数 | 官方附件 1 | 传热模型参数 | 是 | 单位统一、分层读取 |
| 皮肤温度 | 官方附件 2 | 参数标定 | 是 | 插值、误差计算 |
| 换热系数 | O 奖论文 A466 | outstanding 复现对齐 | 否，论文报告结果 | 作为论文复现参数 |

这样写的好处：

- 评委能快速判断数据可靠性。
- 自己也能避免把官方数据、外部数据和论文报告结果混在一起。
- 后续复现代码更容易维护。

## 六、数据可靠性的检查清单

使用任何数据前，都应该检查：

- 是否来自官方或可信公开来源。
- 文件路径和版本是否固定。
- 单位是否统一。
- 时间范围是否一致。
- 空间范围是否一致。
- 缺失值如何处理。
- 是否经过归一化、插值、聚合或筛选。
- 处理后的数据是否保存为可复查表格。

对于外部数据，还要额外说明下载链接或来源名称、下载日期、统计口径，以及和题面数据的匹配方式。

## 七、一句话总结

数学建模不是先选算法，而是先问：

```text
数据从哪里来？
数据是否可靠？
数据能否支撑我的假设、模型和结论？
```

Baseline 可以用粗数据跑通，Advanced 应该尽量使用官方数据做实，Outstanding 则要清楚区分官方数据、外部数据和获奖论文中的表格、参数、结果。
