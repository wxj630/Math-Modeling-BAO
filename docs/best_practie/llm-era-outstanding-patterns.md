# LLM 时代 Outstanding 论文有什么变化

这里的“LLM 时代”主要指 2023、2024、2025 这些赛季之后的论文风格。当前本地已经把 2023-2025 年 MCM/CUMCM ABC 共 18 篇 O 奖复现纳入统一 runner；这篇文档仍然用若干代表题讲共性，不把它写成所有 O 奖论文的逐篇文学评论。

## 一句话结论

LLM 时代的 O 奖论文并没有变成“谁堆方法谁赢”。真正强的论文仍然靠四件事：假设合理、模型贴题、结果可检验、表达清晰。

变化在于：论文更像一个完整计算工作流，文献和方法包装更强，模型链更混合，图表和叙事更精致。但这也带来风险：如果只有漂亮语言和一串算法名，没有可复现结果和检验，反而更容易显得空。

## 和早期 O 奖论文相同的地方

早期优秀论文和 2024 ABC 这些现代样例有共同底层逻辑：

| 评奖维度 | 不变的要求 |
|---|---|
| 假设的合理性 | 假设必须符合现实，并且后续模型真的用到 |
| 建模的创造性 | 创新来自题目机制，不是为了炫算法 |
| 结果的正确性 | 关键数值要可复现、量纲合理、经得起检验 |
| 表达清晰 | 摘要、公式、图表、结论要形成一条证据链 |

所以 O 奖不是 advanced 后面机械加几个模型。更准确地说，O 奖是把题目机制、数据、模型、验证和写作拧成一条线。

## 2023-2025 更明显的变化

### 1. Related Work 和方法框架更重要

现在的优秀论文往往会先说明已有研究如何做，再指出本题特殊之处，最后引出自己的组合模型。

比如 MCM 2024-C 的 2401298 不是直接说“我训练一个模型预测势头”，而是先把 momentum 看成发球优势剥离后的 residual effect，再用 Ljung-Box、Runs Test 和 Dual-Temporal Bayesian Network 解释它。这样“创新”就不是空喊，而是从已有统计检验和 Bayesian network 文献自然过渡出来。

### 2. 模型链更混合

三类题型的边界更模糊：

| 入口题型 | 现代 O 奖常见升级 |
|---|---|
| 动态系统 | 动态仿真 + 生态/经济指标 + 稳定性/灵敏度 + 管理建议 |
| 运筹优化 | 状态估计 + Monte Carlo 情景 + 搜索/路径优化 + Bayesian 更新 |
| 数据建模 | 残差定义 + 统计检验 + 预测模型 + 决策建议/备忘录 |

MCM 2024-A 看起来是生态动态题，但 O 奖论文接了 Lotka-Volterra、Nicholson-Bailey、多样性指数和稳定性分析。MCM 2024-B 看起来是搜索优化题，但先要做动力漂移和不确定性建模。MCM 2024-C 看起来是数据建模题，但最后要给教练策略建议。

### 3. 数据来源交代更细

现在的论文如果只说“我们收集数据”是不够的。需要说明：

- 哪些是竞赛官方数据。
- 哪些是外部公开数据。
- 哪些是论文作者自己设定的场景参数。
- 哪些是 O 奖论文中的结果或校准参数。

MCM 2024-C 是最干净的例子：官方给了 Wimbledon 逐分 CSV，所以 advanced 和 outstanding 都可以围绕官方数据展开。MCM 2024-B 则不同，题面没有事故观测 CSV，O 奖论文需要补 Ionian Sea 深度、洋流、装备性能等外部或场景数据。

### 4. 代码复现和图表工作流更重要

优秀论文越来越像“可复查的计算报告”：

- 输入数据路径清楚。
- 中间表能保存。
- 图表能由代码生成。
- 结果能和论文表格或图形对照。
- 灵敏度分析、稳定性分析或鲁棒性检验能重复。

这正是我们写 B/A/O 教程时要强调的：不是只写论文，也要能把 `result.json`、图表和关键表格追出来。当前 18 篇的统一审计入口见 [outstanding-reproduction-audit-2023-2025-abc.md](./outstanding-reproduction-audit-2023-2025-abc.md)。

## LLM 时代的风险

LLM 能帮助搜索、润色、整理结构，但也容易制造几种“看起来像 O 奖，其实很危险”的论文：

| 风险 | 表现 | 评委为什么不买账 |
|---|---|---|
| 方法堆砌 | 一篇文章塞很多算法，但变量和结论没有连接 | 不像模型链，只像算法清单 |
| 语言漂亮但数值弱 | 摘要很顺，结果没有单位、误差、约束检查 | 经不起检验 |
| Related Work 空泛 | 引一堆文献，却没有说明本题为什么要改造方法 | 看不出创新点 |
| 假设装饰化 | 假设写了很多，后续公式和代码不用 | 假设不服务模型 |
| 图表装饰化 | 图很多，但没有支撑关键结论 | 评委看不到证据链 |

## 2024 ABC 的样例观察

| 题目 | O 奖方法链 | 比 advanced 强在哪里 | 当前复现提醒 |
|---|---|---|---|
| MCM 2024-A | BFM + Logistic + Lotka-Volterra/Nicholson-Bailey + 多样性指数 + 3R 稳定性 | 从“性别比响应曲线”升级为“生命周期、宿主、寄生者、捕食者、稳定性”的生态链 | 已校准为 adaptive 在四个资源情景下均优于 fixed，并保留宿主/寄生者共存 |
| MCM 2024-B | RK4 漂移 + Monte Carlo 粒子 + 熵权装备评分 + Bayesian 搜索更新 + ACO 路径 | 从“位置椭圆和搜索曲线”升级为“后验位置驱动装备和路径”的闭环 | 已用 100000 粒子和检测倍率校准，对齐论文 10 小时 43% 发现概率 |
| MCM 2024-C | 发球校正残差 + EWMA/AUC + Ljung-Box/Runs Test + Dual-Temporal Bayesian Network | 从“势头曲线和逻辑回归”升级为“先定义残差，再证明/检验，再预测和解释”的统计证据链 | 复现使用官方 CSV，方法和结果对齐度最高 |

## 写教程时的判断标准

以后分析 2023-2025 的更多 O 奖论文，可以按这个清单判断：

1. 它有没有清楚的 related work，而不是只列参考文献？
2. 它的创新是不是来自题目机制？
3. 它的模型链是不是前后相接？
4. 它的数据来源是否分清官方、外部、论文参数？
5. 它有没有结果检验、灵敏度分析或稳定性分析？
6. 它的代码复现能否跑出接近论文的关键表格和图？

一句话：LLM 可以让论文更像论文，但 O 奖仍然靠数值可信度和模型贴题度。
