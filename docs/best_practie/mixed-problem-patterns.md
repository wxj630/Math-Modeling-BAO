# 混合题型：真实赛题很少只有一种模型

为了学习方便，我们常把数学建模题分成三类：微分方程、运筹优化、数据建模。但真正的竞赛题往往不是纯题型。获奖论文的优势，常常就在于它能把主模型和后续决策模块接起来。

一句话：

```text
主导题型决定整题的数学基础，混合模块决定论文能不能完整。
```

## 一、三类题型通常怎么混合

### 微分方程题通常不是只有方程

微分方程题的前半段负责模拟系统如何随时间变化，但后半段往往要回答“怎么控制”“怎么设计”“怎么分配资源”。

常见链条：

```text
状态方程 -> 参数拟合 -> 情景模拟 -> 控制策略 -> 资源分配 -> 敏感性/稳定性分析
```

典型例子：

- 疫情传播：`SEIR/SEIQR` 只负责模拟传播，后面还要接隔离策略、药物产能、医疗点选址和药品分配。
- 高温服传热：传热方程只负责预测温度，后面还要接换热系数标定、材料厚度优化和安全约束检查。
- 生态系统：种群动力学只负责演化，后面还要接稳定性分析、干预策略和环境承载力评估。

所以，微分方程题不能只写：

```text
我们建立微分方程并求解。
```

更好的论文结构是：

```text
我们用微分方程模拟系统动态，再基于动态结果设计控制或资源配置方案。
```

### 运筹优化题通常不是只有优化

运筹优化题的核心是决策，但决策模型前面通常要先估计数据，后面还要验证风险。

常见链条：

```text
数据估计 -> 参数校准 -> 优化建模 -> 情景模拟 -> 风险评价 -> 策略解释
```

典型例子：

- 沙漠穿越：不是直接求最短路，而是先确定天气、资源消耗和行动规则，再做动态规划，还要做随机天气情景和失败概率分析。
- 收费站合流：不是只最大化流量，还要估计服务率、排队延误、事故风险、占地面积和自动驾驶比例敏感性。
- 供应链/生产调度：不是只求最优订货量，还要先评价供应商稳定性，再考虑运输损耗、库存风险和鲁棒性。

所以，运筹优化题不能只写：

```text
我们建立目标函数并求最优解。
```

更好的论文结构是：

```text
我们先估计决策参数，再在约束下优化策略，并通过情景模拟和风险评价检验策略稳定性。
```

### 数据建模题通常不是只训练模型

数据建模题的核心是从数据中提取规律，但比赛题最终通常要给出决策、排序、资源配置或政策建议。

常见链条：

```text
数据清洗 -> 特征工程 -> 模型训练 -> 预测/分类/评分 -> 排序/分配/策略 -> 解释与验证
```

典型例子：

- 信贷决策：分类器输出违约概率后，还要决定给谁贷款、贷多少、利率多少、预算如何分配。
- 药物危机：趋势预测后，还要识别传播源头、设计干预策略、评估政策效果。
- 体育奖牌预测：模型预测奖牌概率后，还要输出国家排名、项目布局、教练资源建议。

所以，数据建模题不能只写：

```text
我们训练了随机森林，准确率较高。
```

更好的论文结构是：

```text
我们用机器学习估计风险或潜力，再把模型输出转化为可执行的排序、分配或政策方案。
```

## 二、主导题型和混合模块

建议把每道题拆成两层理解。

| 层级 | 作用 | 例子 |
|---|---|---|
| 主导题型 | 决定整题最核心的数学基础 | ODE、DP、分类器、线性规划 |
| 混合模块 | 让论文从“会算”变成“能决策” | 参数拟合、约束优化、情景模拟、风险评价、敏感性分析 |

这能避免两个常见误区：

- 误区一：看到微分方程题，就只写方程和曲线。
- 误区二：看到数据建模题，就只刷模型准确率。

真正的获奖论文通常是：

```text
一个主模型 + 两三个必要混合模块 + 清楚的验证和表达
```

## 四、为什么混合题型更接近评奖标准

### 假设更容易落地

纯模型容易出现“假设写了但没用”的问题。混合模型会迫使假设进入后续模块。

例如：

```text
药品产能影响恢复率 -> 恢复率影响疫情曲线 -> 疫情曲线影响药品分配
```

这条链条比单独写“假设药品有效”更有说服力。

### 创造性更自然

创造性不是硬套冷门算法，而是把题目机制接起来。

例如：

```text
沙漠穿越 = 路径选择 + 资源消耗 + 天气不确定性 + 风险控制
```

因此 `DP + 情景模拟 + 失败概率` 就是自然创新，而不是算法堆砌。

### 结果更经得起检验

混合模型通常更容易做灵敏度分析或稳定性分析。

例如：

- 改变传播率，看药物阈值是否稳定。
- 改变天气序列，看沙漠路线是否仍可行。
- 改变违约概率，看贷款策略是否大幅变化。
- 改变交通流量，看合流事故率是否突然失控。

这些检验能支撑“结果正确性”，比只给一个最终答案更像获奖论文。

### 文字表述更有主线

混合题型天然适合写成清楚的论文链条：

```text
先模拟现象 -> 再识别关键因素 -> 然后优化策略 -> 最后验证稳定性
```

评委读起来会更顺，因为每一节都回答一个自然问题。

## 五、写作模板

可以用下面的句式把混合题型讲清楚。

### 微分方程题

```text
本文首先建立系统状态方程，用于刻画核心变量随时间的演化；
随后根据模拟结果构建控制/优化模型，确定干预策略；
最后通过灵敏度分析检验关键参数变化下结论的稳定性。
```

### 运筹优化题

```text
本文先从题面数据估计决策参数，再建立满足资源、时间和风险约束的优化模型；
为检验策略可靠性，进一步设置多种情景并进行模拟比较。
```

### 数据建模题

```text
本文先完成数据清洗和特征工程，训练模型得到预测/分类/评分结果；
随后将模型输出转化为排序、资源分配或政策建议，并通过验证指标和敏感性分析检验方案可靠性。
```

## 六、一句话总结

数学建模里的题型分类只是入口，不是终点。

```text
微分方程题后面常接优化、控制、资源分配；
运筹优化题前面常接数据估计、情景模拟、风险评价；
数据建模题后面常接决策、排序、资源配置和政策建议。
```

能把这些模块自然接起来，论文才会从“会算”变成“可信、可用、可获奖”。
