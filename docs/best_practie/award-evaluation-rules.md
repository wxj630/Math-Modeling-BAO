# MCM/ICM 与 CUMCM 评奖规则和专家评阅标准

> 调研时间：2026-07-12  
> 适用范围：COMAP MCM/ICM 与全国大学生数学建模竞赛 CUMCM  
> 核心结论：两类竞赛都没有公开一个永久固定的百分制评分表。公开资料更接近“评审原则 + 奖项档位 + 每题动态细则”。备赛时应把重点放在问题回答、建模合理性、结果可信度、验证分析和论文表达上，而不是机械追求某个算法或图表数量。

## 一、最新奖项比例概览

### 1. MCM/ICM

截至 2026-07-12，COMAP 已公布的最新结果为 2026 年 MCM/ICM，结果发布日期为 2026-05-08。MCM 对应 A/B/C 题，ICM 对应 D/E/F 题。COMAP 使用 designation，即评审等级，而不是严格固定名额制；各奖项比例会随当年参赛质量和评审结果波动。

| 等级 | 含义 | 2026 年合计比例 |
|---|---|---:|
| Outstanding Winner | 最终轮中的顶尖论文 | 41 队，约 0.13% |
| Finalist | 进入最终评审轮的优秀论文 | 452 队，约 1.40% |
| Meritorious Winner | 建模、分析、表达均较优秀 | 2208 队，约 6.85% |
| Honorable Mention | 高于平均水平，过程与表达较扎实 | 7182 队，约 22.30% |
| Successful Participant | 合格参赛或成功参赛 | 21917 队，约 68.04% |
| Unsuccessful / Disqualified / Not Judged | 不充分、违规、未正确提交等 | 合计约 1.28% |

实用口径：

- H 奖及以上约 30.7%。
- M 奖及以上约 8.4%。
- F/O 顶尖档约 1.5%。
- MCM/ICM 的 Meritorious 或 Honorable Mention 不能直接等同于 CUMCM 全国一等奖或全国二等奖，因为评奖池、评阅流程和奖项设计不同。

除常规 designation 外，MCM/ICM 还有专项奖和冠名奖，例如 COMAP Scholarship Award、INFORMS、SIAM、MAA、AMS、ASA，以及 Ben Fusaro、Frank Giordano、Veena Mendiratta、Leonhard Euler、Rachel Carson、Vilfredo Pareto 等问题或学会专项奖。这些奖项通常不是按固定比例发放，而是从顶尖论文中再筛选。

### 2. CUMCM

截至 2026-07-12，CUMCM 最新已完成获奖数据来自 2025 年；2026 年比赛规则已有修订稿，但 2026 年获奖结果尚未产生。CUMCM 分本科组和高职高专组，常规奖项包括全国一等奖、全国二等奖，以及各赛区一、二、三等奖。

| 奖项 | 评奖口径 | 2025 年最新实际比例 |
|---|---|---:|
| 全国一等奖 | 各赛区初评并按名额报送，全国专家组统一评阅后确定 | 本科 294 队，约 0.48%；高职高专 91 队，约 1.33% |
| 全国二等奖 | 各赛区初评并按名额报送，全国专家组统一评阅后确定 | 本科 1204 队，约 1.96%；高职高专 327 队，约 4.78% |
| 赛区一、二、三等奖 | 各省或赛区专家组评阅 | 比例由赛区通知决定，不全国统一 |

2025 年 CUMCM 共有 68311 队参赛，其中本科 61463 队，高职高专 6848 队。全国奖合计 1916 队，整体全国获奖率约 2.80%；其中全国一等奖整体约 0.56%，全国二等奖整体约 2.24%。

## 二、专家到底按什么标准评奖

### 1. 共同原则

MCM/ICM 和 CUMCM 都强调“解决问题的质量”，而不是单独比较模型名字、代码行数或图表精美程度。高奖论文通常满足以下特点：

1. 题目任务回答完整，每个小问都有明确结论。
2. 假设少而必要，且后文确实使用这些假设。
3. 模型从题意自然推导出来，不是把会的算法硬套到题目上。
4. 求解过程可复现，变量、参数、数据、公式和算法步骤说清楚。
5. 结果有单位、量级、对比和现实解释。
6. 有误差分析、敏感性分析、稳定性分析或现实性检验。
7. 图表能帮助评委理解核心结论，而不是装饰。
8. 摘要能快速讲清问题、方法、关键结果和建议。
9. 论文结构清晰，引用规范，无身份信息、抄袭、未披露 AI 使用等违规风险。

一句话概括：高奖论文不是“模型最复杂”，而是“问题答得准、模型讲得通、结果算得对、验证站得住、论文读得顺”。

## 三、MCM/ICM 评阅标准

### 1. 官方公开的评审关注点

COMAP 官方资料没有公开永久固定的百分制评分表。根据官方竞赛说明、FAQ 和历年建议材料，评委主要关注：

| 维度 | 评委实际在看什么 |
|---|---|
| 问题理解 | 是否真正回答题目要求，而不是写自己熟悉但偏题的内容 |
| 建模思路 | 问题拆解、变量定义、模型选择和逻辑链条是否合理 |
| 数学方法 | 方法是否适配问题，推导、计算、算法是否站得住 |
| 假设 | 假设是否合理、必要、有解释，不能把核心困难直接假设掉 |
| 结果 | 是否给出明确结论、建议、数值、比较或解释 |
| 验证 | 是否有误差分析、敏感性分析、稳定性分析、现实性检查 |
| 论文表达 | 摘要、结构、图表、引用、语言和逻辑是否清晰可信 |

### 2. Summary Sheet 的地位

MCM/ICM 非常重视 Summary Sheet。评委会先通过摘要判断论文是否值得继续深入阅读。摘要写得差，即使后文有不错模型，也可能让评委很难快速抓住贡献。

高质量 Summary Sheet 应包含：

- 对问题的简洁理解。
- 采用的核心模型或方法。
- 最关键的数值结果、排序、策略或结论。
- 对模型可靠性的简要说明。
- 对题目所要求的建议、备忘录、报告或政策含义的直接回应。

摘要不适合写成背景介绍，也不适合堆满算法名。它应当像一页浓缩版论文，让评委在很短时间内看出“这队做了什么、算出了什么、为什么可信”。

### 3. 代码是否评分

MCM/ICM 官方明确要求不要提交程序或软件，代码文件不会用于评审。因此，美赛中的代码不是单独评分项。评委无法通过运行代码给分，真正被评阅的是论文中对算法、数据处理、参数、仿真、结果和验证的呈现。

这意味着：

- 代码写得优雅本身不会直接加分。
- 论文中的计算过程必须足够清楚。
- 图表和结果必须能支撑题目要求。
- 若使用外部数据、开源库或 AI 工具，应按规则披露和引用。

### 4. 可视化是否评分

可视化也不是独立的“炫图分”。图表的作用是降低评委理解成本，并让结果、比较、趋势和敏感性分析更可信。

有效图表通常具备：

- 标题、坐标轴、单位、图例完整。
- 只展示和结论有关的信息。
- 正文解释图表说明了什么。
- 图表可以支持模型比较、参数敏感性、策略权衡或最终结论。

无效图表通常包括：

- 只有装饰效果，没有信息量。
- 颜色复杂但没有清晰结论。
- 坐标、单位、样本、变量含义缺失。
- 和正文结论脱节。

### 5. 奖项档位的质量理解

| 奖项 | 可操作理解 |
|---|---|
| Successful Participant | 有认真尝试，但模型、分析、结论或表达存在明显不足 |
| Honorable Mention | 高于平均水平，建模、分析、结论和表达有基本可靠支撑 |
| Meritorious Winner | 多方面优秀，问题回答清楚，模型和结果支撑充分，组织良好 |
| Finalist | 进入最终评审轮，分析完整、逻辑强、清晰易读、全面 |
| Outstanding Winner | Finalist 中的顶尖论文，建模、求解、分析、表达都达到很高水平 |

## 四、CUMCM 评阅标准

### 1. 官方四大标准

CUMCM 官方章程明确写出评奖主要标准：

| 官方标准 | 具体落到论文里 |
|---|---|
| 假设的合理性 | 假设是否符合实际，是否必要，是否用于后续建模与求解 |
| 建模的创造性 | 模型是否贴合题目，有没有合理创新，而不是硬套现成算法 |
| 结果的正确性 | 计算结果是否可复现，量纲是否合理，是否符合现实并经得起检验 |
| 文字表述的清晰程度 | 摘要、结构、公式、图表、结论是否能让评委快速看懂 |

这四条是 CUMCM 最重要的公开评阅原则。很多赛区和教师会把它进一步扩展成“模型、算法、结果、检验、论文”几类，但官方基础口径仍然是上述四项。

### 2. 每题细则如何产生

CUMCM 全国奖项评阅规范说明：正式评阅前，各评阅组会讨论赛题、评阅要点和可能的解题思路，形成可操作的评阅细则；必要时先试评一批论文，再修改细则；之后所有评委按统一细则评阅。

因此，CUMCM 确实存在“评阅细则”，但它通常是按年份、按赛题、按小问动态制定，不是公开固定的长期百分制表格。比如 A 题可能更重视机理模型和误差分析，C 题可能更重视数据清洗、特征构造、模型验证和结果解释。

### 3. 代码与支撑材料

CUMCM 比 MCM/ICM 更明确地把代码和支撑材料纳入规范。2026 年论文格式规范要求附录包含完整可运行源程序，支撑材料也要包含支撑模型、结果、结论的必要材料。

在 CUMCM 中，代码的价值主要是：

- 证明结果不是编造的。
- 证明模型可以复现。
- 支撑数据处理、求解、仿真和图表。
- 保证论文结论与计算输出一致。

代码不是按“软件工程优雅程度”单独评分。更关键的是：程序能运行、结果能对应论文、数据处理过程清楚、关键参数和输出可追踪。如果缺少必要源程序、程序不能运行，或者运行结果与论文明显不符，可能影响评奖甚至取消评奖资格。

### 4. 图表与可视化

CUMCM 论文中的图表同样服务于论证。优秀图表通常出现在这些位置：

- 数据探索：说明数据分布、异常值、缺失值、相关性。
- 模型结构：说明变量关系、流程、状态转移或优化框架。
- 结果展示：说明预测、排序、路径、策略、误差或收益。
- 验证分析：说明敏感性、鲁棒性、残差、对比实验。

图表不是越多越好。评委更喜欢“少而准”的图表：每张图都能回答一个问题，每张表都能支撑一个结论。

## 五、从评委视角反推论文写法

### 1. 摘要

摘要应该是整篇论文的得分入口。建议包含：

- 每个问题的简短处理方案。
- 关键模型或算法。
- 主要数值结果。
- 结论、排序、策略或建议。
- 模型可靠性的简要说明。

不建议摘要只写“本文建立了模型一、模型二、模型三”，却没有关键结果。

### 2. 问题重述与分析

评委需要看到你真的理解题目。问题分析应说明：

- 题目核心矛盾是什么。
- 每个小问之间有什么关系。
- 哪些变量是决策变量，哪些变量是状态变量或观测变量。
- 哪些数据可以直接使用，哪些数据需要处理或补充。

### 3. 假设

好假设通常有三个特征：

- 必要：没有这个假设，模型难以推进。
- 合理：符合现实或题目背景。
- 可追踪：后文真的使用了该假设。

差假设通常把核心问题直接假设掉。例如题目要求预测风险，却假设风险恒定；题目要求优化路线，却假设路线已知。

### 4. 模型建立

模型部分应回答：

- 变量和参数是什么。
- 目标函数或评价指标是什么。
- 约束条件来自哪里。
- 方程、转移规则、损失函数或评分函数为什么合理。
- 模型如何对应题目的每个小问。

模型不一定越复杂越好。复杂模型如果解释不清、验证不足，反而会拖累评分。

### 5. 求解与代码实现

求解部分应说明：

- 使用了什么算法或求解器。
- 输入数据如何进入模型。
- 参数如何估计。
- 计算流程如何复现。
- 输出结果如何转成题目要求的答案。

常见写法示例：

```text
本文将疫情传播过程建模为 SEIQR 常微分方程组，并使用 scipy.integrate.solve_ivp 进行数值积分。
求解器只负责给定参数下的时间推进；模型有效性主要由状态变量定义、参数校准和敏感性分析保证。
```

```text
本文将路径选择问题写成按时间展开的动态规划。状态包含日期、所在节点、剩余资源和可行动作，
转移枚举停留、移动、补给等选择。由于状态空间有限，直接 DP 比通用优化器更便于输出完整策略表。
```

```text
本文从业务数据中提取规模、稳定性、异常率和集中度等特征，
调用 scikit-learn 中的多种分类器估计风险，并用交叉验证和敏感性分析检验模型可靠性。
```

### 6. 结果分析

结果部分不要只贴表。每个结果都应说明：

- 这个数值回答了哪个小问。
- 单位和量级是否合理。
- 与基准方案或已有数据相比如何。
- 结论对现实问题有什么含义。
- 如果参数变化，结论是否稳定。

### 7. 检验与敏感性分析

高奖论文通常不会只给一个结果，而会解释结果为什么可信。常见检验方式包括：

- 和已知数据、基准模型或简单模型对比。
- 改变关键参数，看结果是否稳定。
- 做残差分析、误差分析或交叉验证。
- 检查约束是否满足。
- 讨论模型适用范围和失效情形。

### 8. 模型评价与改进

这一部分不应空泛写“本文模型较好，但仍有不足”。更好的写法是：

- 具体指出模型优点来自哪里，例如结构简单、可解释性强、能复现结果。
- 具体指出局限，例如忽略某类随机扰动、数据时间跨度不足、某些参数来自经验估计。
- 给出可执行的改进方向，例如加入空间相关性、引入随机模拟、改用分层模型、补充外部数据。

## 六、备赛检查清单

提交前可以按下面清单自查：

| 检查项 | 合格标准 |
|---|---|
| 任务完整性 | 每个小问都有明确答案，不遗漏题目要求 |
| 假设 | 少而必要，符合现实，后文实际使用 |
| 变量与参数 | 定义清楚，单位明确，来源可追踪 |
| 模型 | 与题目目标一致，有目标函数、约束、方程或评价指标 |
| 求解 | 算法步骤清楚，参数估计和计算流程可复现 |
| 代码 | CUMCM 需完整可运行；MCM/ICM 不提交代码但论文中要讲清计算逻辑 |
| 结果 | 有数值、有单位、有比较、有解释 |
| 验证 | 有误差、敏感性、鲁棒性、基准对比或现实性检查 |
| 图表 | 每张图表都有信息量，并被正文解释 |
| 摘要 | 一页内说明方法、关键结果和结论 |
| 规范 | 引用完整，AI 使用披露，匿名要求满足，无违规讨论痕迹 |

## 七、两类竞赛的差异

| 维度 | MCM/ICM | CUMCM |
|---|---|---|
| 评奖形式 | designation 档位，比例相对宽 | 全国奖较窄，赛区奖由各赛区决定 |
| 最新结果口径 | 2026 年已公布 | 2025 年为最新已完成结果 |
| 代码提交 | 官方要求不要提交程序或软件 | 要求附录和支撑材料包含完整可运行源程序 |
| 摘要重要性 | 极高，Summary Sheet 是第一入口 | 同样重要，摘要直接影响评委第一印象 |
| 细则公开程度 | 公开评审关注点和奖项描述，不公开固定评分表 | 公开四大原则，每题内部细则动态制定 |
| 论文语言 | 英文 | 中文为主 |
| 违规关注点 | 引用、外部帮助、AI 披露、格式和匿名提交 | 赛中讨论、指导教师介入、AI 披露、支撑材料、相似度等 |
