# Baseline 到 Advanced 到 Outstanding：解法如何逐步变好

数学建模论文不是因为“模型越复杂”就越好，而是因为它更符合题目、更经得起检验、更容易让评委快速看懂。一个好教程应该能说明：同一道题为什么 baseline 只是起点，advanced 为什么更像完整解法，outstanding 为什么接近获奖论文。

数据来源也要同步升级。Baseline 可以先用题面参数跑通，Advanced 应尽量读取官方附件和官方数据，Outstanding 则要清楚区分竞赛官方数据、外部补充数据和 O 奖论文中的参数/结果。更完整的数据说明见 [数据来源实践：模型之前先问数据从哪里来](./data-source-practice.md)。文献综述和 Related Work 的写法见 [Literature Review 实践：不是堆论文，而是解释模型为什么这样建](./literature-review-practice.md)。

可以把三层解法理解为：

| 层级 | 目标 | 典型状态 |
|---|---|---|
| Baseline | 先把题做通 | 有基本模型、能运行、能给出粗结果 |
| Advanced | 把题做实 | 使用真实数据和题面约束，结果可复现、可解释 |
| Outstanding | 把题做成论文级方案 | 模型链完整，有创新、有验证、有清晰表达 |

评奖时真正看的不是代码炫不炫，而是下面四件事：

| 评奖标准 | 评委关心的问题 |
|---|---|
| 假设的合理性 | 假设是否符合实际、是否必要、是否用于后续模型 |
| 建模的创造性 | 模型是否贴合题目，有没有合理创新，而不是硬套算法 |
| 结果的正确性 | 计算结果是否可复现、量纲合理、符合现实、经得起检验，最好有灵敏度分析或稳定性分析支撑 |
| 文字表述的清晰程度 | 摘要、结构、公式、图表、结论是否让评委快速看懂 |

## 一、Baseline：最低可运行解法

Baseline 的任务不是拿奖，而是回答“这道题能用什么模型起步”。它是学习和工程复现的第一层地基。

一个合格 baseline 应该做到：

- 读懂题目主问题。
- 给出最朴素但合理的变量定义。
- 选择一个常见模型族，例如 ODE、线性规划、最短路、回归、分类、聚类。
- 代码能运行，能输出 `result.json`、`report.md` 和基本图表。
- 结果大致量纲正确，不能明显违背题意。

但是 baseline 通常有明显不足：

- 假设偏粗，可能只是为了让模型能跑。
- 数据使用少，可能只用了题面示例参数。
- 约束不完整，很多真实限制还没有进模型。
- 结果缺少验证，只能作为对照组。
- 文字解释像“算法说明”，还不像竞赛论文。

### Baseline 和评奖标准

| 评奖标准 | Baseline 的典型表现 | 常见问题 |
|---|---|---|
| 假设的合理性 | 有基本假设 | 假设太宽泛，没有说明为什么必要 |
| 建模的创造性 | 使用经典模型 | 容易硬套模板，题目贴合度不足 |
| 结果的正确性 | 代码能跑，有初步数值 | 缺少单位检查、敏感性分析和现实校验 |
| 文字表述的清晰程度 | 能说明用了什么方法 | 结构通常松散，摘要和图表不够有说服力 |

### 例子

如果题目是疫情传播，baseline 可以先写 SIR 或 SEIR：

```text
假设人群分为 S、E、I、R 四类，传播率和恢复率为常数，使用 solve_ivp 模拟未来病例变化。
```

这已经能跑，但还不够好。真实赛题往往还要考虑隔离、药物、医疗资源、国家差异和政策干预。

## 二、Advanced：真实数据和题面约束驱动的解法

Advanced 的任务是把题做实。它不只是“换一个更高级算法”，而是把题面中真正重要的约束、数据、目标和评价标准接进模型。

Advanced 通常会比 baseline 多做这些事：

- 读取官方附件或可信数据源。
- 把题面里的容量、时间、成本、温度、安全、概率、预算等限制写成约束。
- 对参数做拟合或校准，而不是拍脑袋给值。
- 输出可复查的中间表、最终表、图和指标。
- 做敏感性分析、误差分析、可行性检查。
- 让后续小问继承前面模型，而不是每问另起炉灶。

### Advanced 和评奖标准

| 评奖标准 | Advanced 的升级 |
|---|---|
| 假设的合理性 | 假设开始和题目语境绑定，并在后续模型中被真正使用 |
| 建模的创造性 | 从通用模型变成题目专用模型，例如把“路径问题”变成“带天气和资源消耗的动态规划” |
| 结果的正确性 | 结果来自真实附件和可复现代码，有约束检查、量纲检查、误差或敏感性分析 |
| 文字表述的清晰程度 | 章节能按问题递进，图表开始服务结论 |

### 例子 1：CUMCM 2018-A 高温作业服

Baseline 可以只做热阻近似：

```text
把多层服装看作串联热阻，估计皮肤温度变化。
```

Advanced 应该进一步：

- 读取官方附件中的材料密度、比热、导热率和厚度。
- 用实测温度曲线标定换热系数。
- 建立随时间变化的传热模型。
- 搜索 II 层和 IV 层厚度，使最高温度和超过 44 摄氏度时间满足约束。

这时模型从“能解释温度变化”升级为“能回答材料厚度怎么设计”。

### 例子 2：CUMCM 2020-B 穿越沙漠

Baseline 可以先找一条最短路：

```text
把地图看作图，边权设为距离，求起点到终点最短路径。
```

Advanced 应该变成动态规划：

```text
状态包含日期、所在节点、剩余水、剩余食物和资金，转移枚举停留、移动、挖矿、补给等行动。
```

因为这道题真正的难点不是“路短不短”，而是天气、资源消耗、赚钱、补给和风险之间的权衡。

## 三、Outstanding：论文级完整方案

Outstanding 不是简单地比 advanced 多堆几个算法。它通常有三个特点：

1. 模型链完整：前一问的变量、参数和结论能自然进入后一问。
2. 创新点合理：创新来自题目本身，不是为了显得复杂而强行套模型。
3. 证据链扎实：结果不仅能算出来，还能解释为什么可信。

Outstanding 解法会特别重视：

- 摘要直接回答题目问题，并给出关键数值结论。
- 假设少而必要，每条假设都服务模型。
- 模型结构贴合真实机制。
- 图表不是装饰，而是支撑结论。
- 对结果做误差、敏感性、鲁棒性或现实合理性检验。
- 明确说明局限性和改进方向。

### Outstanding 和评奖标准

| 评奖标准 | Outstanding 的表现 |
|---|---|
| 假设的合理性 | 假设少、准、必要，并且在模型公式、约束或计算中被使用 |
| 建模的创造性 | 有题目驱动的模型组合或结构创新，例如 ODE + 资源分配、DP + 天气情景、分类器 + 信贷策略 |
| 结果的正确性 | 可复现、可校验、量纲合理，关键结论经得起检验，并有灵敏度分析、稳定性分析或对照实验支撑 |
| 文字表述的清晰程度 | 摘要、图表、公式、结论形成一条清楚证据链，评委能快速抓住贡献 |

## 四、四项评奖标准下的升级路线

### 1. 假设的合理性

Baseline 的假设通常是为了简化问题：

```text
假设传播率为常数。
假设所有车辆服务时间相同。
假设企业违约风险只与信誉评级有关。
```

Advanced 要把假设和数据、约束联系起来：

```text
传播率可被隔离和药物干预改变。
车辆服务时间按收费方式分组。
企业风险由发票稳定性、交易规模、作废率和信誉评级共同决定。
```

Outstanding 要让假设成为模型链的一部分：

```text
隔离率影响 SEIQR 方程中的 I -> Q 转移，药物产量影响恢复率。
收费方式比例影响 M/G/k 排队服务率，并进一步影响合流区负荷。
发票特征进入分类器，分类器输出违约概率，违约概率再进入贷款额度分配模型。
```

判断假设好不好，可以问三句话：

- 这条假设是否符合现实？
- 这条假设是否必要？
- 这条假设后面有没有被模型真正用到？

如果一条假设写在论文里，但后续公式、代码、图表都没有用到，它很可能只是装饰。

### 2. 建模的创造性

创造性不是“用了冷门算法”，而是模型结构贴合题目。

Lit review 在这里很重要。它提供了判断创新点的参照系：先说明已有方法能解决什么，再说明本题多了什么特殊约束，最后说明本文如何改造或组合模型。这样评委才能看出创新来自题目机制，而不是作者临时给算法贴了一个“创新”的标签。

不好的创造性：

```text
看到数据就上随机森林。
看到路径就只做最短路。
看到优化就硬套遗传算法。
```

好的创造性：

```text
疫情题：SEIQR 传播模型 + 药物产能阈值 + 医疗中心覆盖。
沙漠题：动态规划 + 天气情景枚举 + 随机风险模拟 + 博弈策略。
信贷题：发票特征工程 + 多分类器集成 + 贷款额度分配。
```

也就是说，创新应来自题目机制：

- 题目有时间演化，就考虑动态模型。
- 题目有资源约束，就考虑优化或 DP。
- 题目有不确定性，就考虑情景、模拟或鲁棒性。
- 题目有大量数据，就考虑统计学习和验证。
- 题目有决策输出，就把预测结果接到优化或排序中。

### 3. 结果的正确性

Baseline 的结果只要求“能跑出来”。Advanced 和 Outstanding 必须回答“为什么可信”。

结果正确性至少包括：

- 代码可复现：同样输入能得到同样输出。
- 单位合理：时间、金额、温度、概率、流量不能混乱。
- 数值合理：结果不能明显违背常识。
- 约束满足：温度上限、预算上限、容量上限、路径可达性等必须检查。
- 敏感性分析：关键参数变化时，结论不能轻易崩掉。

优秀论文常见写法：

```text
我们首先用附件数据标定模型参数，然后用标定参数进行预测。
对关键参数上下浮动 10% 后，最优策略仍保持稳定，说明模型具有一定鲁棒性。
```

结果正确性不是只放一个最终答案，而是要给评委一条检查路径：

```text
输入数据 -> 参数估计 -> 模型计算 -> 约束检查 -> 结果表 -> 图表 -> 结论
```

### 4. 文字表述的清晰程度

数学建模论文首先是给评委读的。评委没有时间慢慢猜你的模型，所以清晰表达本身就是竞争力。

Baseline 的文字通常只说明方法：

```text
本文采用线性规划求解资源分配问题。
```

Advanced 要说明方法如何服务问题：

```text
本文以企业违约概率和贷款收益为核心指标，建立贷款额度分配模型，在 5000 万元预算约束下最大化银行期望收益。
```

Outstanding 要让摘要和图表直接呈现贡献：

```text
在 5000 万元预算下，模型筛选出 268 家可贷企业，并将其划分为 A/B/C 三类信用等级；
敏感性分析表明，在违约概率整体上升 10% 时，贷款总额分配仍保持稳定。
```

清晰表达的检查清单：

- 摘要是否直接回答每个问题？
- 每一节标题是否对应题目小问？
- 公式前是否定义变量，公式后是否解释含义？
- 图表标题是否能独立说明内容？
- 结论是否给出关键数值，而不是只说“效果较好”？
- 局限性是否诚实说明？

## 五、六道样例题的升级方向

| 赛题 | Baseline | Advanced | Outstanding |
|---|---|---|---|
| MCM 2015-A Ebola | SEIR/SIR 粗传播模型 | 加入国家差异、隔离、药物生产和敏感性 | SEIQR + 药物阈值 + 医疗中心集合覆盖 + 药品分配 |
| CUMCM 2018-A 高温服 | 热阻近似估温 | 读取附件并标定传热参数，搜索厚度 | Crank-Nicholson/传热模型 + 换热系数拟合 + II/IV 层厚度优化 |
| MCM 2017-B Toll Plaza | 简单通行能力估计 | 排队论 + 合流区设计 + 成本/安全指标 | M/G/k 服务水平 + 网络流合流 + 事故率敏感性 + NJ 迁移应用 |
| CUMCM 2020-B 穿越沙漠 | 最短路或粗资源约束 | 按天动态规划，枚举行动和天气 | DP + 1024 情景枚举 + 随机模拟 + 博弈策略 |
| MCM 2019-C Opioid | 线性趋势预测 | 官方 NFLIS/ACS 数据建模，识别高风险县 | 空间传播/Markov 思想 + 源头识别 + 干预策略分析 |
| CUMCM 2020-C 信贷 | 信用评级打分 | 发票特征工程 + 违约风险模型 + 额度分配 | 多分类器 soft voting + 企业分级 + 预算约束下贷款策略 |

## 六、从能跑到能拿奖的实践流程

可以按下面顺序打磨一篇论文。

### Step 1：先做 Baseline

目标是“能跑通”。

- 把题目变量列出来。
- 选一个最朴素模型。
- 写出可运行代码。
- 输出一个初步结果。
- 检查单位和基本合理性。

### Step 2：升级 Advanced

目标是“符合题目”。

- 读取官方附件。
- 用数据估计参数。
- 把题面约束写进模型。
- 输出中间表和最终表。
- 做误差、约束和敏感性检查。

### Step 3：冲 Outstanding

目标是“像论文”。

- 把所有小问连成一条模型链。
- 给出一个自然创新点。
- 用图表解释关键机制。
- 用验证支撑关键结论。
- 摘要中直接写出核心结果。
- 诚实说明模型局限。

## 七、最常见的扣分点

| 扣分点 | 为什么危险 | 如何改 |
|---|---|---|
| 假设很多但后面不用 | 显得空泛，评委会认为模型和假设脱节 | 删除无用假设，只保留会进入公式或约束的假设 |
| 算法很高级但不贴题 | 像硬套模型，创造性反而扣分 | 先解释题目机制，再说明为什么这个模型适合 |
| 只有最终答案，没有检查 | 结果不可验证 | 输出中间结果、约束检查、敏感性分析 |
| 图表很多但没有结论 | 增加阅读负担 | 每张图只回答一个问题，图后必须有一句结论 |
| 摘要只讲方法，不讲结果 | 评委无法快速判断贡献 | 摘要必须包含关键数值、策略和结论 |

## 八、一句话总结

Baseline 解决“会不会做”，Advanced 解决“做得是否符合题目”，Outstanding 解决“评委是否相信并记住你的方案”。

真正的提升不是把算法堆复杂，而是让假设更必要、模型更贴题、结果更可信、表达更清楚。
