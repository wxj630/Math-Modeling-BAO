# Literature Review 实践：不是堆论文，而是解释模型为什么这样建

数学建模论文里不一定会出现一个正式标题叫 `Related Work`。MCM/CUMCM 的获奖论文更常见的写法是把它放在 `Introduction`、`Background`、`Problem Analysis`、`Model Preparation` 或 `References` 里。

但它的作用是一样的：

```text
已有研究怎么做 -> 这些方法解决了什么 -> 本题有什么特殊性 -> 本文为什么这样建模
```

Lit review 不是为了证明“我查了很多文献”，而是为了让评委相信：你的模型选择不是拍脑袋，也不是硬套算法，而是从题目机制和已有方法自然推出来的。

## 一、为什么它直接关系到建模创造性

评奖标准里说“建模的创造性”，但创造性不能只靠作者自己说“本文有创新”。更好的写法是让评委顺着 lit review 自己看出来：

```text
经典方法解决了什么；
经典方法没有覆盖本题的哪个特殊约束；
本文把哪些方法做了题目化改造或组合；
这个改造后面是否真的进入公式、代码、结果和检验。
```

也就是说，lit review 是创新点的参照系。

| 写法 | 评委看到的效果 |
|---|---|
| 只说“本文采用 XX 算法” | 看不出为什么适合本题，也看不出哪里创新 |
| 先讲已有方法，再讲本题新增难点 | 能看出模型选择有依据 |
| 再讲本文如何改造/组合方法 | 能看出创新点来自题目机制，而不是算法堆砌 |
| 最后让创新点进入计算和验证 | 能看出创新不是口号，而是可复现的模型贡献 |

所以，Outstanding 的 lit review 不必很长，但一定要回答：

```text
和已有方法相比，本文到底多处理了什么？
这个多出来的部分，为什么是本题真正需要的？
```

## 二、B/A/O 三层应该怎么写

| 层级 | 文献综述状态 | 写作目标 |
|---|---|---|
| Baseline | 通常不单独写系统综述 | 用一两句话说明采用的是经典模型族 |
| Advanced | 可以补充经典方法背景 | 说明为什么这个模型适合题目，并交代数据/参数如何支撑它 |
| Outstanding | 应该形成小型方法背景链 | 交代已有方法、不足、本题改造点和验证方式 |

### Baseline

Baseline 不需要写成综述论文。它只要说明模型来源属于常见基础模型即可。

示例：

```text
传染病传播常用仓室模型描述人群状态转移，本文以 SEIR 模型作为基础框架。
```

### Advanced

Advanced 要比 baseline 多解释“为什么这个模型适合本题”。

示例：

```text
由于题目关注温度随时间和材料层厚度的变化，本文采用一维非稳态传热模型描述服装层内热量传递，并利用官方实测温度标定换热系数。
```

### Outstanding

Outstanding 要形成更完整的逻辑：

```text
已有方法 A 能刻画核心机制，但没有处理题目中的约束 B；
已有方法 C 能处理决策问题，但需要前置参数估计；
因此本文将 A 和 C 结合，并用数据 D 做标定和检验。
```

这比“本文使用某某高级算法”有说服力得多。

## 三、当前 6 篇 Outstanding 的实践样例

下面按当前选定的 6 篇 Outstanding 论文做拆解。注意：这里的“引用了什么”不是要把参考文献全部搬进教程，而是抓住它们如何支撑模型选择。若 OCR 中个别书名或作者识别有误，以原 PDF 为准。

### 1. MCM 2015-A Ebola：疫情文献引出 SEIQR，再接药物和医疗点优化

论文主要引用了四类资料：

- 疫情事实和参数来源：Baize 等关于 Guinea Zaire Ebola 爆发的 NEJM 论文、Gire 等关于 2014 Ebola 传播起源的 Science 论文、WHO Ebola Situation Report、WHO Ebola 介绍与防护建议。
- 传播模型来源：Chowell 等关于 Ebola 基本再生数和公共卫生措施的研究、Althaus 对 2014 EBOV 再生数的估计、Lekone 和 Finkenstadt 的 Ebola 随机 SEIR 控制干预模型。
- 设施选址来源：Owen 和 Daskin 的 strategic facility location 综述。
- 多目标/多属性决策来源：Belenson 和 Kapur 的多准则线性规划算法、Hwang 和 Yoon 的 Multiple Attribute Decision Making，也就是 TOPSIS 一类方法背景。

它引出模型的方式很清楚：

```text
Ebola 传播已有 SEIR/再生数研究
-> WHO 建议隔离，普通 SEIR 不能表达隔离
-> 扩展为 SEIQR/SEIIR，加入隔离仓室和药物治疗
-> 药物不是无限供应，需要 Logistic 生产增长模型
-> 药物要送到病人，需要医疗中心选址和配送优化
-> 各地区需求不同，用 TOPSIS/多指标评价做药物需求排序
```

所以它的创新点不是“用了 SEIR”，而是把疫情预测、隔离、药物产能、医疗中心选址、药物分配连成一条决策链。Lit review 在这里的作用是：证明 SEIR/SEIQR 有流行病学依据，设施选址和多目标决策有运筹学依据，最后组合模型来自题目真实机制。

### 2. CUMCM 2018-A 高温作业服：传热教材引出 PDE，数值方法引出 Crank-Nicholson

论文主要引用了：

- 传热基础：杨世铭、陶文铨《传热学》，张洪济《热传导》。
- PDE 基础：陆金甫、关治《偏微分方程》。
- 复合介质传热：David W. Hahn 的 Heat Conduction，F. de Monte 关于一维复合介质非稳态热传导的论文。
- 数值方法：Fadugba 等关于 Crank-Nicolson 方法求解抛物型偏微分方程的论文。

它引出模型的方式是：

```text
高温服温度变化本质是多层材料传热
-> 传热学给出热传导方程
-> 作业服可近似为平行无限大平板，只保留厚度方向
-> 复合介质文献给出层间温度连续、热流连续和边界换热条件
-> Crank-Nicholson 适合求解非稳态抛物型 PDE
-> 官方附件缺少换热系数，于是用附件 2 实测皮肤温度反推 h_I 和 h_IV
-> 再把传热模拟接到 II 层、IV 层厚度搜索优化
```

这里的创新点也不是“会写热传导方程”，而是把理论 PDE 变成了可校准、可优化的设计模型：先用附件数据拟合边界换热系数，再用温度约束搜索最小厚度，并做厚度和换热系数的灵敏性分析。

### 3. MCM 2017-B Toll Plaza：交通工程文献引出容量、安全、成本三目标

论文引用的文献可以分成三组：

- 收费站设计和容量：Lin Ruihui 的收费广场规划设计，Zhou Gang 和 Chang Chengli 的收费站通行能力研究，Su Li 的公路收费站设计研究。
- 交通流、排队和延误：Komada 与 Nagatani 的多车道收费站交通流研究，Zarrillo 和 Radwan 的 SHAKER 收费站容量分析，Ayein 等关于收费广场延误估计的方法。
- 安全和仿真：Zhang Zhizhao 关于分流合流区与收费站交通冲突的研究，Abdelwahab 的收费站微观仿真模型，Chau/Xu/Liu、Huang/Huang 关于收费站对交通流影响的模型。

它的引出方式是先把已有研究归类：

```text
已有研究一部分关注通行能力和排队
已有研究一部分关注收费站延误
已有研究一部分关注安全冲突和微观仿真
但题目要求的是收费广场整体设计，不能只看其中一个指标
```

于是论文建立了三段模型：

```text
M/G/K 排队模型 -> 决定收费车道数和服务等级
最大流 + 最小事故风险网络 -> 设计合流路径
安全距离 + 制动距离 + 面积成本 -> 优化收费广场尺寸
```

这个 lit review 最有价值的地方是让“创新点”变得可见：它不是单纯套排队论，而是把收费站设计拆成容量、合流安全、建设成本三个互相制约的目标，再用网络流和优化模型把它们接起来。

### 4. CUMCM 2020-B 穿越沙漠：经典运筹思想引出 DP、随机模拟和博弈

这篇引用的外部文献数量不多，但引用方向很集中：

- 动态规划/运筹基础：《运筹学》教材。
- 随机决策基础：胡奇英《随机运筹学》。
- 博弈论基础：Nash 的 n-person games 均衡论文，以及策略、动态/多阶段 Nash 均衡相关资料。

它引出模型的方式不是“某篇文献直接解决沙漠游戏”，而是从题目机制推出经典模型：

```text
玩家状态可以写成日期、位置、水、食物、资金
-> 后续最优决策只依赖当前状态，具有无后效性
-> 相同状态会重复出现，具有重叠子问题
-> 因此确定天气下适合图动态规划
-> 随机天气下不能直接给唯一最优解，于是用情景枚举、统计分析、随机模拟评价策略
-> 多人局面没有合作机制，因此转为非合作博弈和 Nash 均衡分析
```

这篇给教程的启发是：不是所有 Outstanding 都靠大量相关论文取胜。有些题的 lit review 更像“理论出处说明”。关键是把题目机制和经典理论对上：DP 对应无后效性和最优子结构，随机模拟对应天气不确定性，博弈对应多人竞争。

### 5. MCM 2019-C Opioid：公共卫生背景引出“类疫情”的空间传播模型

论文引用了比较完整的背景和方法链：

- 公共卫生背景：American Society of Addiction Medicine 的 opioid facts，CDC opioid prescribing guideline，Volkow 关于 opioid crisis 和社会经济差异的文章。
- 外部数据：NBER county distance database。
- Markov 和时空传播：Kingman 的 Markov population processes，Barbour 关于 Markov population processes 的极限定理，Gibson 的时空随机模型 MCMC，Yakowitz/Hayes/Gani 的 dynamic Markov fields 与流行病学应用。
- 统计学习：Pearson correlation coefficient，概率机器学习/回归相关资料，R-squared 解释材料。
- opioid 建模和政策：Battista 等 Modeling the opioid epidemic，Pei 等美国流感空间传播预测，Pitt/Humphreys/Brandeau 关于 opioid 公共政策响应建模。

它引出模型的方式是：

```text
opioid 滥用有公共卫生和成瘾机制背景
-> 可以把 opioid 报告数近似看成“感染强度”
-> Markov population / epidemiology 文献说明 Markov 假设可用于人口和流行病动态
-> 县与县之间传播不只看相邻，还受距离影响
-> 用 NBER 县距离数据构造距离加权邻接矩阵 A
-> 用转移矩阵 Q 表示县之间影响强度
-> 社会经济因素不能忽略，用 Pearson 相关筛选特征，再做线性/Logistic 模型
-> 外部空间传播模型和内部社会经济模型组合，识别 epicenters 和 vulnerable counties
```

创新点在于它没有把数据题写成单纯预测题，而是做了两个互补模型：外部模型解释“opioid 在地区之间怎么扩散”，内部模型解释“哪些社会经济特征让某县更脆弱”。最后再通过情景模拟把结果转成政策建议。

### 6. CUMCM 2020-C 中小微企业信贷：信用风险文献引出集成分类，再接贷款组合优化

论文引用的文献主要服务三件事：

- 信用风险度量：王晓燕关于上市民营企业信用风险度量、刘琪关于小额贷款公司个人贷款信用风险评估。
- 机器学习信用识别：顾洲一关于 XGBoost 银行信贷高风险客户识别。
- 集成学习和实现：孟叶等关于基于集成学习的预测方法，王金柱、王翔的 Python 数据分析与挖掘。
- 疫情影响背景：关于个体户、民营企业受疫情影响的调研报告。

它引出模型的方式是：

```text
传统信用风险常用 Logistic/Probit 估计违约概率
-> 但本题没有完整财务报表，主要是发票和交易关系数据
-> 先从企业实力、发展潜力、上下游关系、抗风险能力提取特征
-> 单个分类器可能过拟合或偏弱，因此用 Logistic、Adaboost、GBDT、SVM、随机森林做 Soft Voting
-> 无信贷记录企业没有评级，于是用已有企业训练 XGBoost 多分类预测评级
-> 分类结果不是终点，还要进入贷款额度、利率、风险、客户流失率的多目标规划
-> 疫情场景下再用聚类得到风险乘数，调整信贷策略
```

这篇的创新点在“数据建模 + 决策优化”的连接：机器学习只负责把企业变成违约概率和信誉评级，真正回答题目的是后面的多目标信贷策略模型。Lit review 用信用风险和机器学习文献说明为什么需要分类器，用疫情影响资料说明为什么要引入风险乘数。

这些 Outstanding 的共同点是：文献背景不单独炫技，而是给模型链铺路。

```text
方法背景 -> 模型结构 -> 参数估计 -> 结果验证 -> 策略建议
```

## 四、写作结构模板

一段好用的 lit review 可以按四句话写。

### 第一句：这个领域已有哪类方法

```text
传染病传播研究通常使用仓室模型描述易感者、感染者和康复者之间的状态转移。
```

### 第二句：这些方法能解决什么

```text
这类模型能够刻画疫情随时间演化的基本规律，并便于分析传播率、恢复率和隔离率等参数的影响。
```

### 第三句：本题有什么额外难点

```text
但本题不仅要求预测感染规模，还要求讨论药物生产、医疗中心布局和资源分配策略。
```

### 第四句：本文如何改造或组合模型

```text
因此本文以 SEIQR 模型为主体，并进一步引入药物产能阈值和集合覆盖模型，形成传播预测与干预决策相结合的模型链。
```

这个结构非常适合 MCM/CUMCM，因为它直接服务四项评奖标准：

- 假设合理性：说明假设来自真实机制或已有方法。
- 建模创造性：说明创新是针对本题难点的改造，而不是硬套算法。
- 结果正确性：说明参数、数据和验证怎样接入模型。
- 文字清晰度：让评委先理解“为什么这样建模”，再看公式和代码。

## 五、6 道题的可直接改写句式

### MCM 2015-A Ebola

```text
已有传染病研究常用 SEIR 类仓室模型描述疫情传播，但 Ebola 控制问题还涉及隔离、药物治疗和医疗资源布局。本文在 SEIR 框架上增加隔离仓室和治疗恢复机制，并将药物产能与医疗中心覆盖作为后续优化模块，从而同时回答传播预测和干预决策问题。
```

### CUMCM 2018-A 高温作业服

```text
多层防护服的热安全分析通常依赖非稳态热传导模型，有限差分方法适合处理材料层厚度和边界换热条件变化。本文基于官方实测温度标定换热系数，再搜索满足皮肤温度安全约束的材料厚度，使传热模拟直接服务服装设计。
```

### MCM 2017-B Toll Plaza

```text
收费站通行能力可由排队论估计，但广场改造还必须考虑合流区拥堵、事故风险和自动驾驶车辆比例变化。本文先用 M/G/k 排队模型刻画服务能力，再把车辆流向、合流负荷和事故敏感性接入优化评价。
```

### CUMCM 2020-B 穿越沙漠

```text
资源约束路径问题常用动态规划处理多阶段决策，但本题还包含天气不确定性、补给选择、挖矿收益和对抗情景。本文把日期、位置、资源和资金作为状态变量，并通过天气情景模拟和博弈分析检验路线策略。
```

### MCM 2019-C Opioid

```text
药物危机研究通常结合时间趋势、空间扩散和社会经济风险因子识别高风险区域。本文先清洗 NFLIS 与 ACS 数据形成县级指标，再用空间转移关系描述扩散路径，并据此识别源头区域和干预重点。
```

### CUMCM 2020-C 中小微企业信贷

```text
信用风险评估常将企业经营特征转化为违约概率，再据此制定贷款策略。本文从发票稳定性、交易规模和作废率构造特征，使用集成分类器估计风险，并将预测结果输入贷款额度与利率分配模型。
```

## 六、不要这样写

### 1. 不要堆无关论文

```text
国内外学者对机器学习、神经网络、优化算法进行了大量研究。
```

这句话太空。它没有说明哪些研究和本题机制有关，也没有解释本文为什么采用某个模型。

### 2. 不要只列算法名

```text
本文参考了随机森林、XGBoost、SVM、神经网络、遗传算法和粒子群算法。
```

这样写会让模型像算法拼盘。更好的写法是先说任务需要什么，再说为什么选择某类方法。

### 3. 不要把 lit review 写成百科

数模论文篇幅有限，lit review 不是毕业论文绪论。一般 1 到 3 段即可，重点是把模型选择讲顺。

### 4. 不要引用了却不用

如果文献里提到的方法没有进入公式、代码、结果或验证，就不要放在核心综述里。评委更关心“文献怎么帮助你建模”，不是“参考文献数量多不多”。

## 七、和数据说明分开写

数据说明回答：

```text
数据从哪里来，是否可靠，怎么处理。
```

Lit review 回答：

```text
已有方法怎么做，本文为什么这样建模。
```

两者会互相支撑，但不要混成一节。比如 CUMCM 2018-A：

- 数据说明：官方附件给出材料参数和实测温度。
- Lit review：非稳态热传导和 Crank-Nicholson 适合描述多层材料温度演化。
- 模型设计：用实测温度标定边界换热系数，再优化材料厚度。

## 八、一句话总结

Outstanding 的 lit review 不是“多引用几篇文章”，而是把模型选择讲成一条可检查的理由链：

```text
已有方法能解释核心机制；
本题多了新的约束和决策目标；
本文把方法做了题目化组合；
结果再通过数据和敏感性分析检验。
```
