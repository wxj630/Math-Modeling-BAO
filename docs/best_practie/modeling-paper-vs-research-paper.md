# 数模论文 vs 科研论文：适配比前沿更重要

很多人写数学建模论文时会下意识模仿科研论文：大量相关工作、追最新模型、强调算法复杂度。这个方向并不完全错，但如果不区分两类论文的目标，很容易把数模论文写偏。

一句话区分：

```text
科研论文：在一个窄问题上，把人类知识边界往前推一点。
数模论文：在一个给定现实问题上，用有限数据和合理模型给出可信决策。
```

所以，科研论文更强调新颖性和理论推进，数模论文更强调问题适配、可复现、可检验和可落地。

## 一、两类论文的核心目标不同

| 维度 | 科研论文 | 数学建模论文 |
|---|---|---|
| 主要目标 | 证明一个新方法、新理论或新发现 | 解决题目给定的现实问题 |
| 创新来源 | 原创算法、理论证明、实验超过已有方法 | 经典模型的题目化改造、模型链组合、数据校准、决策输出 |
| 数据要求 | 往往需要完整实验设计、长期观测或公开基准数据 | 通常只能使用题目附件、官方数据和少量外部补充数据 |
| 方法选择 | 可以为了研究问题使用复杂模型 | 必须服务题目任务，不能让复杂度压过可解释性 |
| 结果评价 | 是否超过已有方法，是否可发表 | 假设是否合理、结果是否可复现、结论是否能指导决策 |

这意味着，数模论文不是越复杂越好。模型太复杂，反而可能暴露几个问题：

- 参数估不出来。
- 题目数据支撑不了。
- 代码无法稳定复现。
- 结果无法解释现实。
- 和题目要求的预测、优化、评价、建议对不上。

## 二、O 奖 lit review 需要引用很新的文献吗

不需要刻意追新。O 奖论文真正需要的是引用得准，而不是引用得新。

更好的引用结构是：

```text
经典文献负责证明模型主干可靠；
近期文献负责证明问题背景、数据来源或现实变化；
方法文献负责证明本文采用的算法或指标有依据；
本文创新负责说明怎样把这些方法改造成适合本题的模型链。
```

因此，最新文献可以引用，但不应该为了显得前沿而引用。尤其要避免把自己逼到和真实科研系统硬刚。

例如，一篇最新论文可能使用高频传感器、数字孪生、深度强化学习或大规模现场观测来刻画真实系统。这类方法本身很强，但竞赛题目往往没有对应的数据条件。如果直接引用并声称沿用，评委会自然追问：

```text
这些参数从哪里来？
训练数据在哪里？
为什么你的简化实现还能代表原方法？
结果能不能复现？
是否真的回答了题目任务？
```

所以更稳的写法不是“最新方法不好”，而是：

```text
最新方法能够刻画更复杂的真实系统，但其数据条件、参数条件和研究目标与本题不同。本文在竞赛给定数据和任务约束下，采用更可解释、可复现的经典模型作为主干，并加入题目特有机制完成适配。
```

## 三、MCM 和 CUMCM 的 lit review 有什么差别

MCM 和 CUMCM 都需要文献支撑，但表达习惯不完全一样。

| 方面 | MCM | CUMCM |
|---|---|---|
| 常见位置 | Introduction、Background、Model Preparation、References | 问题分析、模型假设、模型建立、参考文献 |
| 写法风格 | 更像证据链：背景事实、数据来源、模型来源、方法依据 | 更像方法选择说明：题目约束推出模型、算法和检验方式 |
| 外部资料 | 常用国际组织、公开数据库、学术论文、行业报告 | 常用官方附件、统计数据、中文期刊、政策或行业资料 |
| 创新表达 | 强调把多个模型连接成完整解决方案 | 强调针对题目条件做合理假设、推导和求解 |

MCM 更适合写成：

```text
已有研究提供了问题背景和核心模型，外部公开数据补充了题目附件不足，本文再根据任务要求组合预测、优化和检验模块。
```

CUMCM 更适合写成：

```text
根据题目给出的数据结构和约束条件，本文选择某类经典模型，并在此基础上加入不确定性、风险约束或多阶段决策机制。
```

两者的共同点是：lit review 不是堆论文，而是解释“为什么这样建模”。

## 四、最新模型很复杂，怎样引出自己的创新

可以用三步写法。

第一步，承认已有研究的价值：

```text
已有研究已经能够从更细粒度的数据出发，对该类系统进行较完整的建模。
```

第二步，指出本题的限制，不贬低已有研究：

```text
但这些方法通常依赖更完整的参数、长期观测数据或更细的过程记录，而本题只提供有限附件和明确的任务约束。
```

第三步，引出自己的适配创新：

```text
因此，本文不追求完整复刻真实系统，而是以经典模型为主干，结合题目中的 XXX 数据和 XXX 约束，引入 XXX 机制，使模型既能解释主要过程，又能直接回答题目任务。
```

注意，这里的 XXX 必须进入后续模型、代码、结果或检验。否则它只是口号，不是创新。

## 五、可以直接改写的句式

### 动态系统/微分方程题

```text
已有动态系统模型能够描述状态变量随时间的演化，但本题不仅要求预测过程，还要求评估干预策略。本文以经典仓室模型/传热模型/生态动力学模型为主干，结合题目给出的参数和边界条件加入控制变量，并通过灵敏度分析检验结论稳定性。
```

### 运筹优化题

```text
已有优化方法可以处理路径、选址或资源分配问题，但本题中的需求、风险和成本参数并非完全确定。本文先利用题目数据估计关键参数，再建立带约束的优化模型，并通过情景模拟或鲁棒性分析比较不同方案的表现。
```

### 数据建模题

```text
已有机器学习方法能够从多维特征中提取预测信息，但本题的最终目标不是单纯提高预测精度，而是形成排序、配置或政策建议。本文先构造可解释特征并训练预测模型，再将预测结果输入决策模型，使数据建模结果直接服务题目要求。
```

### 通用 lit review 模板

```text
已有方法能够刻画该问题的核心机制，但竞赛题目同时包含数据有限、参数不完全可观测和决策目标明确等特点。若直接引入过于复杂的真实系统模型，容易造成参数不可识别或结果难以复现。因此，本文以经典模型为主干，并结合题目中的 XXX 约束加入 XXX 机制，使模型既保持可解释性和可复现性，又能直接回答题目任务。
```

## 六、不要这样写

不要写成：

```text
已有研究已经很多，本文采用神经网络、遗传算法、随机森林、粒子群算法等方法进行建模。
```

问题是它没有解释每个方法为什么适合本题，也没有说明这些方法之间如何连接。

也不要写成：

```text
最新研究已经提出了非常复杂的真实系统模型，因此本文做了一个简化模型。
```

问题是它像是在承认自己的模型只是低配版本。更好的表达是强调适配：

```text
已有复杂模型为理解真实系统提供了重要参考，但其数据条件和研究目标与本题不同。本文面向竞赛题目中的有限数据和明确任务，构建可解释、可复现、可检验的模型链。
```

## 七、和评奖标准怎么对应

这种写法能同时回应四项常见评奖标准：

| 评奖标准 | 对应写法 |
|---|---|
| 假设的合理性 | 说明哪些真实因素被保留，哪些因为不可观测或影响较小而简化 |
| 建模的创造性 | 说明创新来自题目化改造和模型链组合，而不是硬套算法 |
| 结果的正确性 | 说明数据来源、参数估计、复现方式和灵敏度/稳定性分析 |
| 文字表述的清晰程度 | 让评委先看懂“为什么这样建模”，再进入公式和结果 |

尤其要记住：数模论文里的“经得起检验”，通常不是要求像科研论文那样做多年实验，而是要求在竞赛条件下进行合理的灵敏度分析、稳定性分析、情景对比或结果复现。

## 八、一句话总结

```text
科研论文强调方法前沿性，数学建模论文强调问题适配性。
建模比赛中的创新不一定来自全新算法，而常常来自对经典模型的题目化改造、模型链组合、数据校准和可检验的决策输出。
```
