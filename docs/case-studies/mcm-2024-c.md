# MCM 2024-C 网球势头案例

2024-C 的关键不是只完成第 1 问的势头可视化，而是把“势头定义 -> 随机性检验 -> 换向预测 -> 泛化测试 -> 教练备忘录”串成整题。

赛题页入口：[2024-C Momentum in Tennis](/mcm-track/problems/2024-C)

## 小问递进链

| 小问 | 在整题中的作用 | Advanced 重点 |
|---|---|---|
| q01 | 定义并可视化逐分势头。 | 发球胜率基线、逐分残差、EWMA 势头指数。 |
| q02 | 回应教练“势头只是随机波动”的质疑。 | lag-1 相关、连续得分串、势头范围。 |
| q03 | 预测势头何时换向。 | 未来 8 分换向标签、逻辑回归、关键因素解释。 |
| q04 | 检查模型是否能迁移到其他比赛。 | 31 场比赛泛化、留出比赛、预测质量和局限。 |
| q05 | 把模型结论转成教练建议。 | 临场监控指标、训练建议、备忘录表达。 |

## Baseline 到 advanced 的变化

Baseline 给出每问的模型入口：网络路径、加权评分、趋势预测、报告大纲。Advanced 则读取 COMAP 官方 Wimbledon 逐分 CSV，把同一套势头定义贯穿五问。

| 对照项 | Baseline | Advanced |
|---|---|---|
| 数据 | 题面索引和通用实验表。 | 7284 行 Wimbledon 逐分数据和数据字典。 |
| 主模型 | readiness score 或趋势脚手架。 | 发球校正残差、EWMA 势头、换向预测。 |
| 递进关系 | 每问能单独运行。 | q02-q05 都继承 q01 的势头定义。 |
| 输出 | `experiment_table.csv` 和 baseline score。 | 势头流 CSV/PNG、换向系数、泛化测试、教练备忘录。 |

## Outstanding 预留

- 对不同比赛做交叉验证，检查势头指标是否稳定。
- 把比分压力、破发点、跑动距离和非受迫失误做可解释回归。
- 对 momentum 阈值做敏感性分析。
- 把 q05 写成正式教练备忘录，并统一整题图表风格。
