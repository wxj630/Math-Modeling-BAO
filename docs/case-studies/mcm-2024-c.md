# MCM 2024-C 网球势头案例

这个案例展示 MCM 赛道中从 baseline 到 advanced 的典型升级：同样是 2024-C 第 1 问，baseline 只保留可运行脚手架，advanced 读取官方 Wimbledon 逐分 CSV 并生成势头可视化。

## 题目任务

第 1 问要求建立一个捕捉比赛逐分走势的模型，判断某个时刻哪位球员表现更好、好多少，并给出可视化。

## Baseline 层

路径：

```text
mcm/generic_baselines/reports/2024/C/q01/report.md
mcm/generic_baselines/solutions/2024/C/q01/solution.py
```

Baseline 使用 `network_path_baseline`。它根据题面中的 flow 关键词和真实工作流索引，生成一个透明的 readiness score：

```text
S = sum_i w_i x_i
```

它能回答“这个问题应该从信息流或路径类模型切入”，但还不能回答真实比赛中哪个点出现势头反转。

## Advanced 层

路径：

```text
mcm/question_reports/2024/C/q01/report.md
mcm/question_solutions/2024/C/q01/solution.py
mcm/question_artifacts/2024/C/q01/final_match_momentum_flow.png
```

Advanced 读取 COMAP 官方附件：

```text
docs/mcm-2015-2025/official_assets_extracted/2024/Problem Data- Momentum in Tennis
```

核心升级包括：

- 用发球方得分率建立基线。
- 将逐分结果转成残差。
- 用指数加权移动平均构造 `momentum_p1`。
- 识别强势头换向次数和最大势头点。
- 输出最终比赛的势头流 CSV 与 PNG。

## 从 baseline 到 advanced 的差距

| 对照项 | Baseline | Advanced |
|---|---|---|
| 数据 | 只读题面索引和真实工作流元数据。 | 读取 7284 行 Wimbledon 逐分数据和数据字典。 |
| 模型 | 固定权重 readiness score。 | 发球基线、逐分残差、EWMA 势头指数。 |
| 输出 | `experiment_table.csv` 和 `baseline_score`。 | `final_match_momentum_flow.png`、最大势头点表、换向统计。 |
| 论文可用性 | 适合开题和模型选择。 | 可直接支撑模型建立、结果分析和可视化章节。 |

## Outstanding 预留

后续可补的 outstanding 方向：

- 对不同比赛做交叉验证，检查势头指标是否稳定。
- 把比分压力、破发点、跑动距离和非受迫失误做可解释回归。
- 对 momentum 阈值做敏感性分析。
- 写成完整教练备忘录和论文图注。
