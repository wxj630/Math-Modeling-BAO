# MCM/ICM 2015-C 人力资本网络案例

2015-C 是“以赛题为整体阅读”的典型例子。它不是七个独立小问，而是一条从组织网络建模到 HR 决策报告的递进链。

赛题页入口：[2015-C Managing Human Capital in Organizations](/mcm-track/problems/2015-C)

## 整题主线

| 小问 | 在整题中的作用 | 阅读重点 |
|---|---|---|
| q01 | 建立人力资本网络主模型。 | 岗位层级、部门/办公室结构、填补率、招聘范围、培训依赖。 |
| q02 | 把 q01 的静态网络推进到流失动态。 | 年流失率、中层高流失、生产率损失、网络影响扩散。 |
| q03 | 把流失动态转成两年招聘和培训预算。 | 招聘时间、招聘成本、工资、培训成本、sigma 预算。 |
| q04 | 对更高流失率做可持续性压力测试。 | 25%/35% 年流失、80% 填补率、间接成本。 |
| q05 | 处理中层集中流失和无外部招聘情景。 | 内部晋升链、瓶颈岗位、HR health。 |
| q06 | 把人力资本网络扩展成多层组织网络。 | 信息流、信任、影响力、友谊、团队科学。 |
| q07 | 把前六问收束成 20 页报告和执行摘要。 | 统一假设、模型函数、结果摘要、管理建议。 |

## Baseline 如何读

Baseline 在这里的价值不是给出最终 HR 结论，而是告诉你每一问应该从哪类模型切入：

- q01/q06 用 `network_path_baseline`，说明整题核心是网络结构。
- q02 用 `first_order_dynamic_baseline`，说明需要时间演化。
- q03/q04 用 `resource_allocation_baseline`，说明动态结果要落到预算和容量。
- q05 用 `evidence_table_baseline`，说明情景冲击需要可审计证据表。
- q07 用 `report_outline_baseline`，说明最后一问是论文表达出口。

## Advanced 如何串起来

Advanced 解法把题面参数放进同一条组织模型链：

1. 先用 370 个岗位、46 个 7 人部门、12 个 4 人办公室、85% 填补率等参数建立组织网络。
2. 再把 18% 年流失率和中层高流失转成动态风险与生产率损失。
3. 然后计算两年招聘、培训和恢复空缺所需预算。
4. 接着测试 25%/35% 流失以及中层 30% 离职的压力情景。
5. 最后把团队科学和多层网络作为扩展，并收束成 HR 经理可读报告。

## Outstanding 预留

这道题如果要继续打磨，优先补赛题级内容：

- 统一符号表和岗位层级图。
- 对关键假设做敏感性分析，例如晋升池比例、招聘吞吐量、流失率。
- 把 q01-q06 的图表整理成一套论文叙事，而不是每问各画各的。
- 在 q07 中明确哪些结论来自题面参数，哪些来自建模假设。
